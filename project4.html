Konteks

Latar Belakang & Masalah

Pada era globalisasi, efisiensi sistem distribusi menjadi faktor kunci daya saing perusahaan. Dalam konteks ini, Vehicle Routing Problem (VRP) yang pertama kali diusulkan Dantzig & Ramser (1959) berfokus pada penentuan rute distribusi barang ke sejumlah pelanggan menggunakan armada kendaraan guna meminimalkan biaya operasional.

VRP konvensional hanya mempertimbangkan satu tujuan (biaya/jarak), padahal dunia nyata sering melibatkan banyak faktor yang saling bertentangan. Dari situlah berkembang Multi-Objective VRP (MOVRP), yaitu varian VRP yang mengakomodasi beberapa tujuan sekaligus, seperti meminimalkan total jarak tempuh sekaligus meminimalkan jumlah kendaraan yang digunakan.

Berbagai algoritma telah dicoba untuk MOVRP, mulai dari Genetic Algorithm, Ant Colony Optimization, hingga Hybrid ACO, namun masing-masing masih memiliki keterbatasan seperti konvergensi dini, sensitivitas parameter tinggi, dan beban komputasi besar. Sebagai alternatif, penelitian ini menerapkan Harris Hawks Optimization (HHO): algoritma metaheuristik yang meniru perilaku berburu kooperatif elang Harris, dengan kemampuan menyeimbangkan eksplorasi dan eksploitasi secara adaptif sehingga lebih stabil dan efektif menghindari solusi lokal.

Metodologi

Tahapan Algoritma HHO

1

Input Data & Inisialisasi Parameter

Membaca data pelanggan (koordinat x,y dan permintaan) serta depot dari file .txt. Menginisialisasi parameter: banyak elang (N), iterasi maksimum (T), batas bawah (LB), dan batas atas (UB). Bobot fungsi tujuan ditetapkan w₁ = 0.001 dan w₂ = 100.

Data .txt Kapasitas Kendaraan
2

Pembangkitan Posisi Awal Elang

Setiap elang direpresentasikan sebagai vektor bilangan riil acak berdimensi n (jumlah pelanggan). Posisi awal dibangkitkan secara acak pada interval (0,1) sebagai encoding solusi sebelum diubah menjadi urutan rute.

Random Initialization Rank-based Encoding
3

Decoding: Posisi → Rute Kendaraan

Posisi elang (vektor real) diurutkan berdasarkan ranking untuk mendapatkan urutan kunjungan pelanggan. Urutan ini kemudian dipotong menjadi beberapa rute berdasarkan kapasitas kendaraan, sehingga pelanggan baru dialihkan ke kendaraan berikutnya jika kapasitas terlampaui.

Rank Decoding Capacity Constraint
4

Evaluasi Fungsi Tujuan Gabungan

Menghitung total jarak Euclidean seluruh rute (f₁) dan menghitung jumlah kendaraan yang digunakan (f₂). Fungsi tujuan gabungan dihitung sebagai F(x) = w₁·f₁ + w₂·f₂, dimana w₂ yang besar memastikan minimisasi kendaraan diprioritaskan.

Euclidean Distance Weighted Sum Method
5

Pembaruan Posisi via Eksplorasi & Eksploitasi

Setiap iterasi, energi pelarian mangsa (E) dihitung untuk menentukan fase. Jika |E| ≥ 1, elang melakukan eksplorasi (mencari area baru). Jika |E| < 1, elang mengeksploitasi wilayah sekitar solusi terbaik (X_rabbit) melalui empat strategi besiege. Posisi diperbarui hanya jika menghasilkan nilai F yang lebih kecil.

Eksplorasi Eksploitasi Lévy Flight

Mekanisme HHO

Fase Eksplorasi & Eksploitasi

Algoritma HHO mengadaptasi perilaku berburu elang Harris secara kolektif. Dua fase utama dan empat strategi eksploitasi bekerja bersama untuk menyeimbangkan pencarian global dan lokal.

Fase 1 · |E| ≥ 1

Eksplorasi (Pengintaian)

Elang menjelajahi ruang solusi secara luas, bertengger di lokasi acak atau bergerak menjauh dari X_rabbit untuk menemukan area baru yang menjanjikan.

q ≥ 0.5 · Bertengger acak di dekat elang lain
q < 0.5 · Bergerak menuju rata-rata populasi

Fase 2 · |E| < 1

Eksploitasi (Pengepungan)

Elang mengepung mangsa (X_rabbit) menggunakan empat strategi bergantung pada nilai energi dan probabilitas kabur mangsa (r).

r≥0.5, |E|≥0.5 · Soft Besiege
r≥0.5, |E|<0.5 · Hard Besiege
r<0.5, |E|≥0.5 · Soft Besiege + Lévy Dives
r<0.5, |E|<0.5 · Hard Besiege + Lévy Dives

Cuplikan Kode Python

Program dibangun modular dengan 9 prosedur utama menggunakan Python murni (tanpa library eksternal khusus ML). Encoding solusi menggunakan rank-based decoding untuk mengonversi posisi kontinu menjadi urutan diskrit.

# Konversi posisi elang (real) → urutan pelanggan → rute
def decode_rank(posisi_elang):
    n = len(posisi_elang)
    rank_pos = [0] * n
    for j in range(n):
        rank = 1
        for l in range(n):
            if posisi_elang[l] < posisi_elang[j]:
                rank += 1
        rank_pos[j] = rank
    return rank_pos

def bentuk_rute_berdasarkan_kapasitas(urutan, permintaan, kapasitas):
    rute, rute_sekarang, muatan = [], [], 0
    for idx in urutan:
        if muatan + permintaan[idx] > kapasitas:
            rute.append(rute_sekarang)
            rute_sekarang, muatan = [idx], permintaan[idx]
        else:
            rute_sekarang.append(idx)
            muatan += permintaan[idx]
    if rute_sekarang: rute.append(rute_sekarang)
    return rute
